# Releases

This folder contains release-facing documentation that remains relevant across iterations.

## Contents

- `RELEASE_CHECKLIST.md`
- `RELEASE_0.8.0_RC_NOTES.md`

## Scope

Keep this folder limited to:

- release checklists
- release notes
- changelog-style release artifacts

Do not store dated operational evidence here. Use `docs/operations/evidence/` for that material.
